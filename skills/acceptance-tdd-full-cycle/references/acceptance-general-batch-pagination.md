# 通用验收：批量与分页

---

## 1. 是否分页与契约

| 验收点 | Then 判据示例 |
| --- | --- |
| 默认分页参数 | `page`/`pageSize` 缺省与上限符合 spec；**无**单次返回超大包拖垮 |
| 总条数 / 是否有下一页 | 与真实数据一致；空列表边界文案与状态码 |
| 排序稳定性 | 同排序键重复时二级排序（如 id）明确，避免分页间「跳行」「重复」 |

---

## 2. 分页边界值

| 验收点 | Then 判据示例 |
| --- | --- |
| `page=0`、负数、超大页码 | 按 spec 纠偏或错误；**无**未定义行为或空指针 |
| `pageSize` 超上限 | 截断或拒绝，与文档一致 |
| 最后一页不足一页 | 条数正确；再请求下一页为空或 404 策略一致 |
| 并发增删导致页漂移 | 用户可理解行为（提示刷新 / 游标分页）；**须在 spec 或 Then 写明** |

---

## 3. 新旧数据兼容性

| 验收点 | Then 判据示例 |
| --- | --- |
| 新字段缺省 | 旧客户端读新 API：缺省/null 展示与 spec 一致 |
| 旧数据无新字段 | 新 UI 不崩溃；默认值或「-」占位 |
| 枚举扩展 | 未知枚举值：前端降级展示；服务端拒绝写非法值 |
| 迁移后 ID/外键 | 关联查询正确；**无**孤儿记录暴露为错误权限数据 |

---

## 4. 批量写与幂等

| 验收点 | Then 判据示例 |
| --- | --- |
| 批量导入部分失败 | 回滚整批 vs 部分成功：与 spec 一致；错误行可定位 |
| 重复提交同一导入文件 | 幂等或明确重复报错；**无**双倍库存/双倍扣款 |

---

## 5. 与分领域验收的关系

迭代涉及 **订单 / 库存 / 交易**：批量扣减、部分发货等与 `acceptance-domain-order-fulfillment.md`、`acceptance-domain-inventory-warehouse.md` 互链，避免重复或矛盾
