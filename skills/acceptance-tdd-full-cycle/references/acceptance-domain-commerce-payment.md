# 领域验收：交易与支付

**适用**：spec / 迭代涉及「交易 / 支付」时用本文查漏 **Then**；与 [acceptance-domain-order-fulfillment.md](acceptance-domain-order-fulfillment.md)、[acceptance-domain-inventory-warehouse.md](acceptance-domain-inventory-warehouse.md) 等 **做交叉引用**，同一判据只在一处写全。

---

## 1. 金额与币种

| 验收点 | Then 判据示例 |
| --- | --- |
| 金额精度 | 金额计算无精度丢失；小数位符合业务规范 |
| 币种与汇率 | 多币种交易汇率生效；汇率波动不影响已下单金额 |
| 金额分摊 | 多商品/优惠金额分摊正确；无分摊错误导致的异常 |

---

## 2. 支付流程与状态机

| 验收点 | Then 判据示例 |
| --- | --- |
| 支付方式切换 | 支持的支付方式可正常使用；不支持的方式被拒绝 |
| 支付回调与通知 | 支付成功/失败回调处理正确；状态同步无延迟 |
| 超时与重试 | 支付超时订单关闭；重试支付不重复扣款 |

---

## 3. 退款与冲正

| 验收点 | Then 判据示例 |
| --- | --- |
| 全额/部分退款 | 退款金额正确；订单状态同步；无重复退款 |
| 退款时效与路径 | 退款原路返回；时效符合平台规范 |
| 冲正与对账 | 冲正操作正确；对账无差异 |

---

## 4. 幂等与防重

| 验收点 | Then 判据示例 |
| --- | --- |
| 幂等键与防重 | 重复提交同一订单/支付请求被拒绝；幂等键生效 |
| 并发下单 | 同一用户并发下单无超卖；库存与订单状态一致 |

---

## 5. 对账与审计

| 验收点 | Then 判据示例 |
| --- | --- |
| 对账流程 | 订单、支付、退款数据与第三方平台一致；对账无差异 |
| 审计日志 | 交易操作可追溯；关键字段不泄露敏感信息 |
