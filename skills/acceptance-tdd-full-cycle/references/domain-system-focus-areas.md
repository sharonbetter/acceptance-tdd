# 领域系统形态（验收用备忘）

本文件在本技能包内自治使用，**不依赖**其它目录。

---

## 用途

回答两件事：**本迭代是否命中下表中的任一业务形态（如交易/支付、订单履约）？** 若是，**该形态特有的**风险与 Then 判据应写在哪里、如何与 **验收场景** 互链？

- 与验收场景主表（结构见 `atdd-checklist-template.md`）**互链**：本备忘定「形态与专属面」，场景表里用一句话指到本文对应小节即可，避免两处各写一套矛盾 Then。
- **分工**：
  - 本文件：**领域形态**
  - `cross-cutting-coverage-dimensions.md`：**横切质量与体验（DIM+UX）**
  - 通用功能验收（登录、注册、权限、分页、搜索、页面、手势等）：见 `acceptance-dimensions-general-index.md`

---

## 1. 判定：本迭代是否命中下列形态

对下表逐行勾选：命中 / 不适用。若**全部不适用**，仍建议在验收文档中保留一行 `N/A` 并写清理由（例如：纯配置脚本、无对业务态）。

| 形态 | 命中时至少须覆盖（示例方向） | 详细验收点（本包 references） |
| --- | --- | --- |
| **交易 / 支付** | 金额精度、币种、幂等键、失败回滚、对账字段、敏感数据不落日志 | `acceptance-domain-commerce-payment.md` |
| **订单履约** | 状态机、超时取消、部分发货、库存扣减与覆盖边界、售后入口 | `acceptance-domain-order-fulfillment.md` |
| **库存 / 仓储** | 并发扣减、预占与释放、盘点差异、多仓路由 | `acceptance-domain-inventory-warehouse.md` |
| **权限 / 多租户** | 租户隔离、角色变更生效点、越权与 IDOR、审计字段 | `acceptance-domain-permissions-multitenancy.md` |
| **订阅 / 计费** | 周期边界、试用转正、退款与冲正、发票字段 | `acceptance-domain-subscription-billing.md` |
| **消息 / 异步** | 延迟与幂等、失败重试、重试与补偿、幂等键 | `acceptance-domain-messaging-async.md` |
| **运营后台 / 配置** | 发布与生效、灰度、误操作回滚、批量导入校验 | `acceptance-domain-ops-config.md` |
| **广告 / 推荐** | 频控、素材合规、空结果降级、计费曝光（若命中须与统计口径对齐） | `acceptance-domain-ads-recommendation.md` |

团队可增删行；**未列出但业务明显属于表中某一类**时，**自行加行**并在下文「领域形态小节」展开。

## 2. 领域形态小节怎么写（与验收场景对齐）

- **一行摘要**：本迭代中的形态 + 最关键的 1~3 条**领域风险（可收用语）**。
- **场景映射**：仅对 **命中** 的形态写节；每条子节下列 **Given/When/Then** 或指向验收场景中已编号的场景 ID。
- **N/A**：无命中形态时，写明「无对业务态 / 无领域扩展」，**不留空**。

---

## 3. 与验收场景的互链写法（示例）

在验收场景表「备注 / 领域」列写：`见领域形态 <交易/支付>` 或 `领域形态 N/A (CLI 工具)`。
在本文件对应小节写：`场景：S-12、S-13` 等反向引用。

---

## 4. 常见错误

- 把**通用四维度（正常/异常/边界/非法）**塞进领域形态小节撑篇幅 — 四维度应在验收场景表，本文只收**领域形态专属**。
- 领域备忘与场景表**长期不同步** — 改任一须同步另一侧或显式标注「已废弃」。
