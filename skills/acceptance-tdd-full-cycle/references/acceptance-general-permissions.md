# 通用验收：权限管理

**适用**：RBAC、ABAC、资源级权限、菜单/按钮级控制、API 与 UI 双通道。

---

## 1. 权限控制有效性（正向）

| 验收点 | Then 判据示例 |
| --- | --- |
| 各角色默认能力 | 角色 A 可访问的菜单/API 与 spec 矩阵一致；**禁止**能力缺项未测 |
| 细粒度资源权限 | 仅能操作本人/本部门数据时：列表、详情、写接口均受限 |
| 权限变更后生效 | 重新登录或刷新权限后 UI 与 API 同步；若 spec 为「下次请求生效」则 Then 写明 |

---

## 2. 水平越权（IDOR / 错用他人 ID）

| 验收点 | Then 判据示例 |
| --- | --- |
| 篡改 URL/请求体中的资源 ID | 返回 403/404（与 spec 一致）；**无**返回他人资源内容 |
| 批量接口中的混入 ID | 部分无权限时：整批失败或仅成功有权限子集（**必须与 spec 一致**） |
| 导出/下载链接带 token | token 与资源绑定；换用户 token 不可下载他人文件 |

\*待确认：404 vs 403 对外策略（防枚举 vs 明确拒绝）。

---

## 3. 垂直越权（低权限执行高权限动作）

| 验收点 | Then 判据示例 |
| --- | --- |
| 普通用户调用管理端 API | 拒绝；**无**仅依赖前端隐藏按钮 |
| 重复利用旧管理 token | 角色降级后旧 token 不可再执行管理操作 |
| 工作流跳过步骤 | 无权限状态不可「直接 PATCH 到终态」 |

---

## 4. 安全与审计（与多租户领域文档可叠加）

| 验收点 | Then 判据示例 |
| --- | --- |
| 敏感操作留痕 | 审计字段或审计日志含操作者、对象、时间（以合规要求为准） |
| 租户隔离 | 租户 A 的 token **不可**读租户 B 数据（见 `acceptance-domain-permissions-multitenancy.md`） |

---

## 5. 落表提示
- **验收场景表**：按「角色 × 资源 × 动作」矩阵拆行，Then 写**具体 HTTP/UI**，避免「权限正确」空话。
- **E2E**：至少覆盖「无权限用户尝试」与「权限变更后」各一条关键路径
