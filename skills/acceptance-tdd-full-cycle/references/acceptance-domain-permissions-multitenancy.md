# 领域验收：权限与多租户

**适用**：多租户隔离、租户内权限控制、角色变更生效、越权防护等场景。

---

## 1. 租户隔离

| 验收点 | Then 判据示例 |
| --- | --- |
| 数据隔离 | 租户 A 不可访问租户 B 的数据；API 层与 DB 层隔离一致 |
| 租户标识传递 | 租户 ID 不可伪造；跨租户请求被拒绝 |
| 租户创建/删除 | 租户创建后数据独立；删除后数据清理符合规范 |

---

## 2. 角色与权限

| 验收点 | Then 判据示例 |
| --- | --- |
| 默认角色 | 新建用户默认角色与 spec 一致 |
| 角色变更生效 | 角色变更后权限即时或按时效（按 spec）生效 |
| 超级管理员 | 超级管理员权限与 spec 一致；不可超越职责范围 |

---

## 3. 越权防护

| 验收点 | Then 判据示例 |
| --- | --- |
| 资源级越权 | 篡改资源 ID 不可访问他人资源；返回 403/404 |
| 垂直越权 | 低权限不可执行高权限操作 |
| 批量越权 | 批量操作中混入跨租户/跨用户 ID 被拒绝 |

---

## 4. 审计与日志

| 验收点 | Then 判据示例 |
| --- | --- |
| 权限变更审计 | 权限变更记录可追溯；含操作者、被操作者、时间 |
| 敏感操作留痕 | 关键业务操作含租户/用户标识；可定位问题 |
