# 通用验收：登录与会话

**适用**：会话过期、多设备登录、单点登录、登出等场景。

**待确认**：过期时长、滑动窗口 vs 固定过期、多设备策略。

---

## 2. 登录前：未登录态一致性

| 验收点 | Then 判据示例 |
| --- | --- |
| 多入口进入同一受保护 URL | 未登录表现一致（同一重定向目标或同一错误码/文案） |
| 未登录可调用的公开接口 | 与 spec 白名单一致；**不**泄露登录才可见的数据（字段裁剪、列表为空 vs 403 须与 spec 一致） |
| 登录前缓存/本地草稿（若有） | 与登录后合并或丢弃规则明确；**无**静默以他人身份提交 |

---

## 3. 登录后：登录态一致性

| 验收点 | Then 判据示例 |
| --- | --- |
| 同一会话多 Tab / 多请求 | 权限与用户信息一致；无「A Tab 已登出 B Tab 仍显示已登」除非 spec 允许并最终收敛 |
| 登出后 | 全客户端会话清除；回退前**不**恢复已认证页面内容（或按 spec 仅展示脱敏态） |
| 角色/权限变更后 | 已持有旧 JWT 是否即时失效（按 spec）；若延迟生效须在 Then 写明可观察窗口 |

---

## 4. 传输与存储安全（可观察 proxy）

| 验收点 | Then 判据示例 |
| --- | --- |
| 凭证传输 | 生产环境登录请求走 **HTTPS**；**Then** 可写「网络层由部署保证」+ 集成校验重定向到 HTTPS（若适用） |
| 密码与令牌不落日志 | 响应/前端控制台/服务端日志抽样：**无**明文密码、完整 refresh token（按团队清单） |
| 记住我 / 设备绑定 | Cookie 属性（HttpOnly、Secure、SameSite）符合 spec；敏感操作二次验证 |

**说明**：「是否明文传输」在自动化中通常体现为 **HTTPS + 不在 body 以外渠道泄露**；若仅 HTTP 环境则验收文档须标 **环境假设** 或 **缺陷/风险**。

---

## 5. 密码设定与校验（与注册文档衔接）

| 验收点 | Then 判据示例 |
| --- | --- |
| 登录失败次数与锁定 | 错误提示**不**区分「用户不存在」与「密码错误」**若 spec 要求防枚举**；锁定与解锁可观察 |
| 弱密码策略 | 若登录页允许设密/改密：与 `acceptance-general-auth-register.md` 中策略一致 |

---

## 6. 与验收场景 / E2E 的映射提示

- **E2E**：过期时间可配合时钟 mock 或缩短 token TTL 的测试环境。
- **集成**：`Set-Cookie` / `Authorization` 头、`401/403` 与 body 契约。
- **勿**把「测 mock 调用次数」当 Then；应断言 **HTTP/UI/后续请求是否被拒**
