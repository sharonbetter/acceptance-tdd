# 通用验收：搜索与筛选

**适用**：关键字搜索、筛选器、排序、聚合查询、保存搜索条件。

---

## 1. 单一条件与单关键词

| 验收点 | Then 判据示例 |
| --- | --- |
| 精确匹配 vs 模糊 | 行为与 spec 一致；大小写、全半角规则写明 |
| 无结果 | 空列表 + 约定文案/占位；**无** 500 或泄露堆栈 |
| 特殊字符与注入防护 | `\`、`%`、`_`、emoji、超长字符串：不崩溃；语义符合 spec（转义/拒绝） |

---

## 2. 组合条件

| 验收点 | Then 判据示例 |
| --- | --- |
| 两两组合 | 与/或逻辑与 spec 一致；**建议**用真值表或矩阵行落在验收场景表 |
| 互斥条件 | 同时选中时：拒绝或自动纠正，提示清晰 |
| 默认值叠加 | 进入页默认筛选选项与 URL 参数同步（若支持 deep link） |

---

## 3. 排序正确性

| 验收点 | Then 判据示例 |
| --- | --- |
| 各排序键 | 升序/降序首屏与翻页后全局有序（在数据固定 fixture 下断言） |
| 可排序字段类型 | 数字/日期/字符串排序规则正确；**无**字符串排序当数字用导致 `'10' < '2'` |
| 排序与分页结合 | 翻页无重复/遗漏（同 `acceptance-general-batch-pagination.md`） |

---

## 4. 性能与体验（与横切维度「体验（有 UI）」配合）

| 验收点 | Then 判据示例 |
| --- | --- |
| 防抖/节流 | 快速输入不产生风暴请求（可观察请求数或 loading 态） |
| 结果延迟展示 | loading 与取消旧请求行为符合 spec |

---

## 5. 落表提示
- **验收场景表**：每个「组合」至少一行 Then 指向**固定数据集**下的期望 ID 列表或 count。
- **勿**只写「搜索结果正确」；写清 **排序字段 + 顺序 + 数据集版本**
