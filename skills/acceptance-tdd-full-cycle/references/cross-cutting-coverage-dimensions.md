# 横切覆盖维度（DIM）与体验（摘要）

本文件在本技能包内自治使用，**不依赖**其它目录。

---

## 用途

在写 **验收场景** 与 **Then** 时，除业务功能外，对照下列 **横切维度** 查漏；有 UI 时须覆盖 **体验**（可用「`UX`」标签行或写明 **刻意不测 + 原因**）。

与 **领域形态** 分工见 [domain-system-focus-areas.md](domain-system-focus-areas.md) 开篇：**领域形态**管「是否命中表中交易/支付、订单履约等**任一行**」；**本文**管「不按表中某一行单独展开，但多数系统都适用的横切点」。

---

## 建议维度（按需勾选）

| 维度 | 验收提示（Then 须可观察或注明专项） |
| --- | --- |
| **逆向 / 撤销** | 用户撤销、管理员回滚、补偿路径与数据一致 |
| **多入口** | 同一能力从菜单/深链/推送进入，权限与状态一致 |
| **时区 / 数字 / 本地化** | 展示与存储约定、夏令时边界、千分位与小数分隔 |
| **体验（有 UI）** | 加载/空/错态、可达性焦点、关键路径耗时感知、错误可恢复 |
| **观测与运维** | 关键业务失败可定位（日志/指标字段不泄露密钥） |

将上述缺口落场时，仍须满足 **Given / When / Then** 与 `atdd-checklist-template.md` 中的闸门要求
