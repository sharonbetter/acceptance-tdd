# 通用验收：注册与开户

**适用**：自助注册、手机/邮箱验证、邀请码、企业开户、批量开通等。

---

## 1. 账号不可重复性

| 验收点 | Then 判据示例 |
| --- | --- |
| 同一键重复提交 | 返回 spec 约定错误码/文案；**无**第二条活跃用户或**无**覆盖静默成功（以 spec 为准） |
| 大小写/空白规范化 | 按 spec 规定 `User` 与 `user` 等价：行为一致；若区分：Then 写明 |
| 并发双请求同账号 | **无**双插入；最终仅一条成功或明确幂等（依赖 DB 唯一约束 + 应用层提示） |

\*待确认：唯一键范围（全局 / 租户内）、软删后是否允许复用用户名。

---

## 2. 注册信息验证正确性

| 验收点 | Then 判据示例 |
| --- | --- |
| 必填项缺失 / 类型错误 | 字段级错误与 HTTP 状态符合 API 契约；前端与后端规则**一致**（或文档声明仅服务端为准） |
| 邮箱/手机格式 | 非法格式拒绝；合法边界格式（+号、国际区号）按 spec |
| 验证码 / 邀请码 | 错误、过期、一次性使用；重放同一验证码行为明确 |
| 服务条款 / 年龄门（若有） | 未勾选不可提交；勾选状态持久化符合 spec |

---

## 3. 密码设定安全性（注册/首次设置）

| 验收点 | Then 判据示例 |
| --- | --- |
| 长度与字符类 | 低于下限拒绝；符合下限即可通过（**Then** 写清具体下限与类要求） |
| 常见弱密码字典（若启用） | `123456`、与用户名相同等被拒绝 |
| 确认密码一致 | 不一致时拒绝提交；**无**仅前端校验而后端未校验 |
| 密码哈希 | 响应与日志中**无**明文密码；存储为单向哈希（由架构/代码审查或专项测证明） |

---

## 4. 注册后状态与登录衔接

| 验收点 | Then 判据示例 |
| --- | --- |
| 需激活流程 | 未激活不可登录或仅受限权限；激活链接一次性/过期 |
| 自动登录 | 若 spec 注册成功即登录：首屏与 `token` 与「手动登录」一致 |
| 审计 | 注册事件可追溯（时间、来源 IP/设备若合规允许） |
